# Business Design Frameworks

Frameworks for designing new businesses, products, and customer experiences.

## Business Model Canvas

**Origin**: Alexander Osterwalder, 2008

**Purpose**: Visualize and design business models on a single page.

### The 9 Building Blocks

#### Right Side (Value Creation & Delivery)

**1. Customer Segments**
Who are we creating value for?

Types:
- Mass market
- Niche market
- Segmented
- Diversified
- Multi-sided platforms

**2. Value Propositions**
What value do we deliver to the customer?

Elements:
- Newness
- Performance
- Customization
- Design
- Brand/Status
- Price
- Cost reduction
- Risk reduction
- Accessibility
- Convenience

**3. Channels**
How do we reach our customers?

Phases:
1. Awareness
2. Evaluation
3. Purchase
4. Delivery
5. After-sales

**4. Customer Relationships**
What relationship does each segment expect?

Types:
- Personal assistance
- Dedicated personal assistance
- Self-service
- Automated services
- Communities
- Co-creation

**5. Revenue Streams**
For what value are customers willing to pay?

Types:
- Asset sale
- Usage fee
- Subscription
- Lending/Renting/Leasing
- Licensing
- Brokerage fees
- Advertising

#### Left Side (Efficiency)

**6. Key Resources**
What do we need to deliver our value proposition?

Types:
- Physical
- Intellectual
- Human
- Financial

**7. Key Activities**
What must we do to deliver our value proposition?

Types:
- Production
- Problem solving
- Platform/network

**8. Key Partners**
Who are our key partners and suppliers?

Motivations:
- Optimization
- Risk reduction
- Resource acquisition

**9. Cost Structure**
What are the most important costs?

Characteristics:
- Cost-driven
- Value-driven
- Fixed costs
- Variable costs
- Economies of scale
- Economies of scope

### How to Use the Canvas

1. **Start with Customer Segments** - Who are you serving?
2. **Define Value Propositions** - What problem do you solve?
3. **Map Channels & Relationships** - How do you reach and keep customers?
4. **Identify Revenue Streams** - How do you make money?
5. **List Key Resources & Activities** - What do you need and do?
6. **Add Key Partners** - Who helps you?
7. **Calculate Cost Structure** - What does it cost?
8. **Iterate** - Test assumptions, refine

## Design Thinking

**Origin**: IDEO, Stanford d.school

**Purpose**: Human-centered approach to innovation and problem-solving.

### The Five Stages

```
     ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
     │         │    │         │    │         │    │         │    │         │
     │EMPATHIZE│───▶│ DEFINE  │───▶│ IDEATE  │───▶│PROTOTYPE│───▶│  TEST   │
     │         │    │         │    │         │    │         │    │         │
     └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
           │                                                           │
           └───────────────────── ITERATE ─────────────────────────────┘
```

### Stage Details

#### 1. Empathize

**Goal**: Understand users deeply

**Methods**:
- User interviews
- Observation
- Immersion
- Journey mapping

**Questions**:
- What do users say?
- What do users do?
- What do users think?
- What do users feel?

#### 2. Define

**Goal**: Frame the right problem

**Tools**:
- Point of View (POV) statement
- How Might We (HMW) questions
- Persona creation

**POV Template**:
```
[USER] needs [NEED] because [INSIGHT]

Example:
"Busy working parents need quick healthy meal solutions
because they feel guilty about feeding their kids fast food
but don't have time to cook."
```

**HMW Questions**:
```
How might we... help busy parents prepare healthy meals quickly?
How might we... reduce guilt about meal choices?
How might we... make cooking feel less time-consuming?
```

#### 3. Ideate

**Goal**: Generate many possible solutions

**Methods**:
- Brainstorming
- Worst possible idea
- SCAMPER
- Mind mapping
- Crazy 8s

**Brainstorming Rules**:
1. Defer judgment
2. Encourage wild ideas
3. Build on others' ideas
4. Stay focused on topic
5. One conversation at a time
6. Be visual
7. Go for quantity

#### 4. Prototype

**Goal**: Build to think and test

**Principles**:
- Start rough
- Fail fast
- Build to learn
- Low fidelity first

**Prototype Types**:
- Paper prototypes
- Storyboards
- Role-playing
- Digital mockups
- Physical models

#### 5. Test

**Goal**: Learn from user feedback

**Methods**:
- Usability testing
- A/B testing
- Feedback sessions

**Questions to Ask**:
- What worked?
- What didn't work?
- What questions do you have?
- What would you change?

## Customer Journey Mapping

**Purpose**: Visualize the complete customer experience across all touchpoints.

### Journey Map Structure

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      CUSTOMER JOURNEY MAP                                │
├─────────────┬─────────────┬─────────────┬─────────────┬────────────────┤
│   STAGE     │  AWARENESS  │CONSIDERATION│  PURCHASE   │   RETENTION    │
├─────────────┼─────────────┼─────────────┼─────────────┼────────────────┤
│   ACTIONS   │             │             │             │                │
├─────────────┼─────────────┼─────────────┼─────────────┼────────────────┤
│ TOUCHPOINTS │             │             │             │                │
├─────────────┼─────────────┼─────────────┼─────────────┼────────────────┤
│  THOUGHTS   │             │             │             │                │
├─────────────┼─────────────┼─────────────┼─────────────┼────────────────┤
│  EMOTIONS   │             │             │             │                │
├─────────────┼─────────────┼─────────────┼─────────────┼────────────────┤
│ PAIN POINTS │             │             │             │                │

│OPPORTUNITIES│             │             │             │                │
└─────────────┴─────────────┴─────────────┴─────────────┴────────────────┘
```

### How to Create a Journey Map

1. **Define the persona** - Who is the customer?
2. **Define the scenario** - What goal are they trying to achieve?
3. **List the stages** - What phases do they go through?
4. **Map touchpoints** - Where do they interact with you?
5. **Capture emotions** - How do they feel at each stage?
6. **Identify pain points** - Where are they frustrated?
7. **Find opportunities** - How can you improve?

## STP (Segmentation, Targeting, Positioning)

**Origin**: Philip Kotler

**Purpose**: Define target market and competitive positioning.

### The STP Process

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│                 │     │                 │     │                 │
│  SEGMENTATION   │────▶│   TARGETING     │────▶│  POSITIONING    │
│                 │     │                 │     │                 │
│  Divide market  │     │ Select segments │     │ Define position │
│  into segments  │     │   to focus on   │     │  in customers'  │
│                 │     │                 │     │     minds       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 1. Segmentation

**Segmentation Variables**:

| Type | Variables | Examples |
|------|-----------|----------|
| **Geographic** | Region, city, climate | Urban/Rural, North/South |
| **Demographic** | Age, gender, income, education | Millennials, High-income |
| **Psychographic** | Lifestyle, values, personality | Health-conscious, Tech-savvy |
| **Behavioral** | Usage rate, loyalty, benefits sought | Heavy users, Brand loyal |

### 2. Targeting

**Targeting Strategies**:

| Strategy | Description | When to Use |
|----------|-------------|-------------|
| **Undifferentiated** | One offer for entire market | Commodity products |
| **Differentiated** | Different offers for different segments | Resources available |
| **Concentrated** | Focus on one segment | Limited resources |
| **Micromarketing** | Individual customer tailoring | High-value customers |

**Segment Evaluation Criteria**:
- Size and growth potential
- Structural attractiveness (competition)
- Company fit (resources, capabilities)

### 3. Positioning

**Positioning Statement Template**:
```
For [TARGET SEGMENT]
Who [NEED/WANT]
[PRODUCT/BRAND] is a [CATEGORY]
That [KEY BENEFIT]
Unlike [COMPETITORS]
[PRODUCT/BRAND] [KEY DIFFERENTIATOR]
```

**Example**:
```
For busy professionals
Who need to stay productive on the go
MacBook Air is a premium laptop
That combines power with extreme portability
Unlike other ultrabooks
MacBook Air offers seamless integration with iPhone and Apple ecosystem
```

                        │
   LOW ─────────────────┼───────────────── HIGH

        ○ Budget        │         ○ Value
                        │
